# Hackathon Sponsor Submissions

> 1-hour hackathon project templates showcasing the **latest features** from each sponsor's December 2024 changelog.

## Quick Start

1. Choose a submission folder
2. Copy `.env.example` to `.env` and add your API keys
3. Install dependencies (`pip install -r requirements.txt` or `npm install`)
4. Run the demo (`python main.py` or `npm run dev`)

Each submission follows this structure:

```

├── main.py             # Main code (Python projects)
├── index.js            # Main code (JS projects)
├── requirements.txt    # Python dependencies
├── package.json        # Node dependencies
└── .env.example        # API key template
```

## How to Use These Submissions

### For Hackathons
1. Pick a sponsor whose prize you want to target
2. Use the template as your starting point
3. Customize for your specific use case
4. Make sure to highlight the **latest feature** in your submission

### For Learning
1. Each project demonstrates best practices for that API
2. Code is intentionally simple and well-commented
3. Extend the demos to build more complex applications

### For Multi-Sponsor Submissions
Combine multiple templates! See `sponsor_combinations_matrix.md` for synergy ratings.

Example combinations:
- OpenAI + Pinecone + Vercel = RAG Chat App
- Anthropic + Supabase + Clerk = Protected AI Dashboard
- Gemini + Neon + Streamlit = AI Data Explorer
- AWS + MongoDB + Stripe = Serverless SaaS
- LangChain + Qdrant + Gradio = AI Search Interface
- CrewAI + HuggingFace + Streamlit = Multi-Agent Research Tool
